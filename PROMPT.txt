在 CSV output 的基础上新增 JSON output 通道：用户传 --output 时根据文件后缀(.csv/.json/.jsonl)自动选择序列化格式，jsonl 每行一条 step 记录包含 timestamp 字段(取自 dool_time plugin, 没有 time plugin 时退化为单调递增 counter,从启动秒起算),json 顶层为 samples 数组,每条记录保留 plugins 各自的列名作为 key 嵌套在 plugin 名下(比如 cpu: {usr, sys, idl, wai, hiq, siq})而不是把所有列平铺到顶层,嵌套时要处理 plugins 名称带特殊字符(-- 号、空格)的 escape 规则跟 JSON 规范一致;CSV 和 JSON 同一次 run 互斥,后指定的覆盖前一个,提示信息走 stderr。

在 Diskset 基础上新增 NetSet:用户用 --netset 名字:eth0,eth1,wlan0 把一组网卡绑成别名,后续 --net -N 别名 调用时合并该集合的实时速率并按 alias 顺序输出,每行的列头要带原始接口名(eth0_rx,eth0_tx,... 而不是一个聚合数字),列宽按最长原始名算;netset 名称空间跟 diskset 独立但解析时复用 getopt 同一段循环,二者都未指定时插件按 -N 或 --process-count 的字面参数处理;netset 配置文件从用户家目录下 .dool 子目录里的 netsets.conf 启动时自动加载,文件每行一条 alias:eth0,eth1 格式以 # 开头行视为注释,加载失败(例如文件不存在或单行格式错乱)不报错仅 stderr 打 warning,命令行 --netset 参数覆盖同名的 conf 加载项。

在 plugin 加载基础上新增 Plugin Health Check:启动时对每个被选中的 plugin 跑一次 self-test(调用 plugin.extract() 一次,判断返回值是否抛异常、是否返回非 dict、是否返回的 dict key 数量跟 vars() 里头的 base 变量对不上),任一 plugin 自检失败就在 stderr 打 [plugin-fail] <name>: <reason> 警告并从该次 run 的 plugins 列表移除而不是直接崩整个 dool,自检通过率低于 50% 时额外打一行 [plugin-fail] <N> of <M> plugins unhealthy, output may be incomplete 提示;用户传 --strict-plugin 时任一失败直接 exit 2,不做降级;插件列表里既有内置 plugin 也有用户 .dool 子目录下的扩展 plugin,二者的异常要区分来源(file path 在 warning 里带出来,内置标 (builtin),扩展标 .dool 子目录下对应文件名)。

在 Profiling mode 基础上新增 CSV 与 JSON profiling:用户传 --profile 同时 --output 时 profiling report 末尾追加一段 [profile] 段,csv 末尾加两行(列名 profile_total_ms 和 per_plugin_ms,值以 plugin name 为 key 时 per_plugin_ms 只能塞在 csv 最后一个 plugin 的扩展列里,前面的列要补空字符串对齐),jsonl 末尾每条加 _profile: {plugin_name: ms} 嵌套对象,json 文件额外写顶层 profile 字段跟 samples 同级;profile 关闭(--profile 不传)时 profiling 代码路径整个不执行(连 time.time() 调用都没有),打开时累计多个 plugin 的总耗时和单 plugin 耗时在 loop 结束才聚合,聚合时除以总 step 数取平均值。检查修复一下。
